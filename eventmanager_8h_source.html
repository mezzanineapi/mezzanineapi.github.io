134  /// @class EventManager
135  /// @headerfile eventmanager.h
136  /// @brief This is a container for Events and facilitates the transfer of data.
137  /// @details The Event Manager Exists to passed important information about
138  /// Gamestate from where it is generated to where it is needed. It is the Game
139  /// Developers option whether they want to grab events directly using the get
140  /// functions that have filters, or if they want to get all the events at once
141  /// from a central location and dispatch form there. \n
142  /// Since all User input comes in the form of events, this is also where user
143  /// input Polling and optional input sources like Joysticks are controlled
144  /// from. \n
145  /// All of these event are stored in an internal Queue and order is preserved.
146  /// So the First item In will be the First Out (FIFO). This is not strictly a
147  /// FIFO buffer, there are a number of functions for getting of managing
148  /// specific kinds of events. Generally these 'Filtered' management functions
149  /// Still return the first of those kinds of event.
150  /// @warning Delete pointers you get from this. Anything can create events and
151  /// Put them here, and anything can get them out, This means the simple way to
152  /// not cause memory leaks is to have the routines extracting the events delete
153  /// the events.
154  /// @warning Currently this is not thread safe, even though it should be.
155  ///////////////////////////////////////////////////////////////////////////////
156  class MEZZ_LIB EventManager: public ManagerBase, public Singleton<EventManager>
157  {
158  private:
159  /// @internal
160  /// @brief All the internal data for this EventManager
161  Internal::EventManagerInternalData* _Data;

164  /// @brief Checks for quit messages and adds them to the queue

169  /// @details Since copying, or having more than one event manager doesn't seem to make sense
170  /// I just made it non-copyable.
171  EventManager(const EventManager& Dummy)
